Labels Gmail existants

Architecture technique

── Tables Supabase ──
user_labels : id, user_id, name, description, prompt, color, position, gmail_label_id, is_default, created_at
→ Chaque label du user avec son prompt IA et le lien vers le label Gmail
email_classifications.label_id : FK → user_labels.id
→ Remplace l'enum classification_result (A_VOIR/FILTRE/BLOQUE)
── Classification pipeline ──
1. Charger les labels du user : SELECT * FROM user_labels WHERE user_id = ?
2. Assembler le system prompt dynamiquement depuis les labels
3. LLM reçoit la liste des labels + descriptions → choisit le meilleur
4. Résultat = label_id (pas un enum fixe)
── Onboarding scan ──
1. Gmail API labels.list → labels existants + couleurs
2. Gmail API messages.list par label → compter les emails
3. Échantillonner 2-3 emails par label pour les exemples (metadata only)
4. Fusionner avec les 7 defaults Kyrra → proposer au client
